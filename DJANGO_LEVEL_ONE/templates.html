The template contains the static parts of an html page (parts that are always the same).
We have template tags, which have their own special syntax that allows you to inject dynamic content.

First you need to create a templates directory and then a subdirectory for each specific app's templates. It saved in top level directory of your project:

  • first_project/templates/first_app
    • To let know Django about templates we need to edit DIR key inside of TEMPLATES dictionary in the settings.py.

    To dynamically generate correct file path string use Python's os module

    We will use this os module to feed the path to the DIR key inside of the TEMPLATES dictionary

    In settings.py we have a variable BASE_DIR = os.path.dirname(os.path.dirname(os.path.abspath(__file__))) that contains path to our project's directory
    Well to join our templates directory to the project we need to add this string in settings.py: TEMPLATE_DIR = os.path.join(BASE_DIR,"templates").
    Below we have to find variable TEMPLATES and add our new variable "TEMPLATE_DIR" in 'DIRS': [TEMPLATE_DIR,],

    After it we can create index.html file in new directory "templates" with common html code inside.
    Now we can add som "dynamic" part inside our HTML - adding some template's tag inside HTML code: {{ insert_me }}
    Than inside our application directory changing views.py:
    def index(request):
        my_dict = {'insert_me':'Hello I am from view.py!'}
        return render(request,'index.html',context=my_dict)
    Now if we will update the page we saw our page, from our template/index.html and string inside this page from view.py: 'Hello I am from view.py!'
