For working with Django recommended use venv. In this project we will work with Anaconda env. To create new venv use this command:
conda create --name myDjangoEnv python=3.5.4 django=1.11.8
By default Anaconda installing the latest version of package.

To show your environment use conda info --envs
To activate environment use source activate myDjangoEnv
To create new Django project use command django-admin startproject first_project
Python create folder with name of your Django project in root directory with some init files.

__init__.py

This is a blank Python script that let's Python know that this directory can be treated as a package

urls.py

This is a Python script that will stor all the URL patterns for your project.Basically the different pages of your web application.

wsgi.py

This is a Python script that acts as the Web Server Gateway Interface.It will later on help us deploy our web app to production.

manage.py

This is a Python script that we will use a lot. It will be associates with many commands as we build our web app!

When we run python manage.py runserver we start our webserver with initial page that can be reached by http://localhost:8000

Django project consists of applications that perform a particular functionality and can be reused.

To create some application use command python manage.py startapp first_app. This command also create new directory with files.

admin.py

In this file you can register your models which Django will then use them with Django's admin interface

apps.py

Here you can place application specific configurations

models.py

Here you store the application's data models

tests.py

Here you can store test functions to test your code

views.py

This is where you have functions that handle requests and return responses

Migrations folder

This directory stores database specific information as it relates to the models
